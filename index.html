KeepOn

A Flutter reminder app for time-bound tasks that should keep resurfacing until they are finished or expired.

Watch demo View technical integration

Application Outline

Built to keep unfinished work visible.

KeepOn focuses on the gap between writing a task down and actually returning to it. The app combines a lightweight task list with repeated local reminders, quiet hours, progress feedback, and simple activity-aware timing.

Focus

Next task at a glance

The home screen highlights the most urgent active task with a remaining-time label and progress bar.

Reminders

Persistent local alerts

Notifications are scheduled while a task is active and stop when the user completes or deletes the task.

Control

Quiet hours and settings

Users can adjust reminder intervals, notification behaviour, and rest periods without leaving the app.

UX Journey

From intention to completion.

The interaction model was designed around a short loop: capture the task, keep it visible, remind at useful moments, then remove it from attention when it is done.

01 Launch

A short branded splash screen introduces the app before moving to the task list.

02 Capture

The user adds title, notes, start time, and end time for a time-bound task.

03 Prioritise

The home screen surfaces active tasks and visually signals urgency.

04 Remind

Local notifications repeat while the task remains active and incomplete.

05 Adjust

Settings let the user choose reminder interval, quiet hours, and notification state.

06 Finish

Completion removes the task from the active list and cancels future reminders.

Data Handling

Local-first by design.

KeepOn stores task and settings data locally on the user's device. The app does not require an account or remote server to operate, which keeps the data model small and transparent.

Task data

Title, optional description, start time, end time, completion state, and task identifiers are persisted with Hive.

Preference data

Reminder interval, notification switch, and quiet-hour preferences are saved with SharedPreferences.

Sensor data

Accelerometer readings are used for live stationary detection and are not stored as a long-term personal data record.

Technical Integration

APIs and services that enhance the app.

KeepOn uses Flutter plugins and device APIs to move beyond a static task list and support reminders that respond to time, settings, and device context.

Notifications

flutter_local_notifications

Schedules repeated task reminders locally and cancels them when tasks are no longer active.

Time

timezone

Supports consistent reminder scheduling by calculating notification times with timezone-aware values.

Sensors

sensors_plus

Reads accelerometer data so the app can detect stationary device moments for activity-aware timing.

Storage

Hive

Stores task records locally with a small, fast persistence layer suited to offline-first use.

Settings

SharedPreferences

Persists lightweight user preferences such as quiet hours and notification options.

Architecture

Riverpod services

Separates task state, notification logic, sensor logic, storage, and settings into maintainable modules.

Design Process

Designed from problem framing to implementation.

Problem

Tasks disappear from attention

The project began with the observation that dismissed reminders often fail to keep unfinished work present.

Storyboarding

A short repeatable loop

The storyboard centred on create, resurface, adjust, and complete, keeping the app narrow and usable.

Wireframes

Home first

The interface prioritised a clear home screen, a focused editor, and settings that directly affect reminders.
